# Admin Panel Setup Guide

A hidden admin panel at `/admin` lets you (and only you) edit any article — title, body, image, tags, everything — from a browser. Changes appear on the live website **immediately, with no rebuild step**.

## Architecture at a glance

```
Browser (admin)  ──→  Convex DB  ──→  Browser (visitor)
       ↑                                    ↑
       └── you edit here              └── blog pages fetch from Convex live
```

- **Source of truth: Convex DB.** Articles live in a Convex `articles` table.
- **Frontend reads live:** `/blog` and `/blog/:slug` query Convex at runtime.
- **No GitHub commits, no Vercel rebuilds** when you edit. Save = instant.
- **Image storage:** Convex's built-in File Storage (no Supabase, no external service).

### Trade-off you should know about

Blog pages used to be prerendered at build time (SSG), so AI crawlers (GPTBot, ClaudeBot, etc.) saw the full content even without running JavaScript. With Convex-backed live edits, blog pages **render client-side**. AI crawlers may see less content per article. The landing pages (`/`, `/dakiraty`, `/quran`, `/about`) are still SSG'd — only `/blog/*` is dynamic.

If AI search becomes critical later, the fix is to add server-side rendering for `/blog/:slug` routes using Convex's server SDK. We can do that as a follow-up.

## One-time setup (15 minutes)

### Step 1 — Initialize Convex in the repo

In the repo's root directory, run:

```bash
cd /path/to/dakiraty-repo
npm install
npx convex@latest dev
```

When prompted:
- Log in to Convex (it'll open a browser — use your existing account)
- Pick "Create a new project"
- Name it: `dakiraty` (or whatever)
- Pick the free `dev` deployment

Convex will:
- Generate `convex/_generated/` (do NOT edit; do NOT commit if it appears — actually you should commit it, but Convex CLI handles that)
- Deploy the schema, functions, and storage config
- Write `.env.local` with `CONVEX_DEPLOYMENT` and `VITE_CONVEX_URL`

Leave `npx convex dev` running in a terminal while you test locally. For production, you'll use `npx convex deploy` (described below).

### Step 2 — Set admin credentials in Convex

Open the Convex dashboard (`https://dashboard.convex.dev`), navigate to your project, click **Settings → Environment Variables**, and add:

| Variable | Value |
| --- | --- |
| `ADMIN_EMAIL` | `tahabot1999@gmail.com` |
| `ADMIN_PASSWORD` | `tybpro123` *(or pick a stronger one — see security note below)* |

These are read by `convex/auth.ts` at login time. They're encrypted at rest by Convex, never exposed to client-side code.

⚠️ **Security note:** `tybpro123` is a weak password. Anyone who finds the `/admin` URL can brute-force it in seconds. Recommend: replace with a 16+ character random string like the kind generated by `openssl rand -base64 24`. You can update `ADMIN_PASSWORD` anytime; the change takes effect immediately, no redeploy needed.

### Step 3 — Wire Convex to Vercel

In your Vercel dashboard for the `website-dakiraty` project:

1. **Settings → Environment Variables → Add:**
   - Name: `VITE_CONVEX_URL`
   - Value: paste the `VITE_CONVEX_URL` from your `.env.local` (looks like `https://something-something.convex.cloud`)
   - Environments: Production, Preview, Development
2. **Deployments → Redeploy** the latest commit to pick up the env var.

### Step 4 — Deploy Convex to production

Convex `dev` runs against a dev deployment. For production, run once:

```bash
npx convex deploy
```

This deploys your `convex/` directory to a production deployment. The CLI shows the production URL — make sure it matches your `VITE_CONVEX_URL` in Vercel (or update Vercel to use the prod URL).

> **Tip:** Add a Vercel build step. In Vercel project settings → Build & Development Settings → set `Build Command` to:
> `npx convex deploy --cmd 'npm run build'`
>
> This deploys Convex automatically every time you push to GitHub.

### Step 5 — Migrate your existing 25 articles into Convex

Right now your 28 articles (25 AR + 3 FR) live as `.md` files in `content/blog/` and `content/blog-fr/`. They need to be imported into Convex one time.

1. **Log in to the admin panel** to create a session token:
   - Visit `https://website-dakiraty.vercel.app/admin` (or `localhost:3000/admin` locally)
   - Log in with your `ADMIN_EMAIL` + `ADMIN_PASSWORD`
   - After login, open browser DevTools → Application → Local Storage → copy the value of `dakiraty_admin_token`

2. **Run the migration script:**

```bash
ADMIN_TOKEN=paste_your_token_here \
VITE_CONVEX_URL=https://your-deployment.convex.cloud \
npm run convex:migrate
```

You'll see output like:
```
📚 Reading Arabic articles from content/blog/...
   Found 25 Arabic articles
📚 Reading French articles from content/blog-fr/...
   Found 3 French articles

🚀 Importing 28 articles into Convex...
   Batch 1: +10 new, 0 skipped
   Batch 2: +10 new, 0 skipped
   Batch 3: +8 new, 0 skipped

✅ Migration complete.
   Inserted: 28
   Skipped (already in DB): 0
```

After this, all 28 articles live in Convex and the blog pages will load them.

## Daily usage

1. Go to `https://website-dakiraty.vercel.app/admin`
2. Log in with your email + password
3. Click any article in the list to open the editor
4. Edit title, excerpt, body (Markdown), tags, image, etc.
5. Upload a new image: click "رفع صورة" → file picker → uploaded to Convex storage → image URL automatically updated
6. Click **حفظ** (Save). Your change is live on the website **immediately** — no rebuild, no wait.
7. Click "عرض على الموقع" to open the live page in a new tab.

The admin panel:
- ✅ Is **not in the navbar** — only accessible by visiting `/admin` directly
- ✅ Is **not in the sitemap**
- ✅ Is **blocked by robots.txt** (`Disallow: /admin`)
- ✅ Has `<meta name="robots" content="noindex, nofollow">` on every admin page
- ✅ Sessions last 7 days, then you log in again

## Updating the password / email

Change `ADMIN_EMAIL` or `ADMIN_PASSWORD` in the Convex dashboard env vars. New value takes effect immediately on next login — no redeploy, no code change.

If you want to **log out everyone** (e.g. you suspect the token leaked), use the Convex dashboard → Data → `sessions` table → delete all rows. Anyone with an old token gets booted on their next request.

## What about the daily content agent?

The daily scheduled agent (`Dakiraty Daily Content - 5 Arabic Articles`) still writes `.md` files to `content/blog/` via GitHub commits. Those new articles won't appear on the live site until you also import them into Convex.

Two options going forward:

**Option A (simple):** After the daily agent runs each morning, run `npm run convex:migrate` from any machine. It's idempotent — skips articles already in Convex, imports only new ones.

**Option B (automated, future work):** Update the daily agent's prompt to call `convex/articles.bulkImport` directly instead of writing `.md` files. Then the agent writes straight to Convex, no migration step needed. This is a Phase 5 task — happy to do it when you're ready.

For now, the daily agent and admin panel are decoupled: agent writes `.md`, admin edits Convex. Migrate manually as needed.

## Troubleshooting

**"Invalid credentials" on login**
- Double-check `ADMIN_EMAIL` and `ADMIN_PASSWORD` in Convex dashboard env vars
- Make sure you re-deployed Convex after setting them (`npx convex deploy`)

**Blog pages show "جاري التحميل..." (loading) forever**
- `VITE_CONVEX_URL` isn't set in Vercel, or doesn't match your Convex deployment
- Check browser DevTools → Network for `convex.cloud` requests; they should return 200

**Image upload fails**
- Convex File Storage requires the active deployment to have storage enabled (it's on by default)
- Check browser DevTools → Network for the upload request status

**"Unauthorized" on save**
- Your session expired (7-day TTL). Log out and log back in.

**Migration script "Cannot find module 'convex/_generated/api.js'"**
- You haven't run `npx convex dev` yet, so the generated files don't exist. Run that first.
